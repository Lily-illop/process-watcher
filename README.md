# Process Watcher — мониторинг аномалий процессов в Linux

##Запуск
Для авто запуска на **Linux** можно прописать 
```
crontab -e
# Добавить строку:
*/15 * * * * /home/your_path_to/ProcWatch/proc_watcher.sh
```

## Проблематика

Пользователи Ubuntu сталкиваются с необъяснимыми тормозами и нагревом из-за «сбесившихся» процессов (утечки памяти, бесконечные циклы, процессы-зомби). Решения вроде `top`/`htop` требуют ручного вмешательства в терминал и не умеют проактивно оповещать.

**Цель:** создать фонового агента на Bash, который автоматически детектирует аномалии и присылает уведомления (всплывающие окна, Telegram).

## Функциональные требования

### Минимальная версия (MVP)

| Требование | Описание |
| :--- | :--- |
| **Мониторинг процессов** | Чтение `/proc` или `ps` (PID, имя, время жизни, RSS память, % CPU) |
| **Детектирование** | Превышение времени жизни (порог в часах) и абсолютного лимита памяти (в МБ) |
| **Уведомления** | `notify-send` + запись в лог-файл (`~/.local/share/proc_watcher/log`) |
| **Игнор-лист** | Список имён процессов (через `IGNORE_COMMS` в конфиге), которые не проверяются |
| **Конфигурация** | Файл `proc_watcher.conf` с параметрами: `MAX_AGE_HOURS`, `MEMORY_LIMIT_MB`, `CHECK_INTERVAL_SEC`, `IGNORE_COMMS` |
| **Режимы запуска** | Однократная проверка (`./proc_watcher.sh`) и режим демона (`--daemon`) |
| **Обработка сигналов** | Корректное завершение по `SIGTERM`/`SIGINT` |

### Максимальная версия (расширенная)

**Всё из минимума плюс:**

1. **Анализ роста памяти** — разница между замерами за N минут (например, +100 МБ за 10 минут)
2. **Аномалии CPU** — текущее потребление >50% при обычном <5%
3. **Ротация логов** — по размеру (10 МБ) или через `logrotate`
4. **Systemd-юнит** — автозапуск через `systemctl --user`
5. **Кроссплатформенность** — поддержка Windows (WSL или Git Bash) через определение `OSTYPE`
6. **Тестирование** — минимальный набор тестов (`test.sh` с имитацией аномалий)

## Нефункциональные требования

| Категория | Требование |
| :--- | :--- |
| **Язык** | Bash 4+ |
| **Зависимости** | Только стандартные (`ps`, `grep`, `awk`, `bc`, `notify-send`, `logger`, `curl` — опционально) |
| **Производительность** | Потребление CPU <5% одного ядра |
| **Портативность** | Работает на Ubuntu 20.04–24.04, поддержка WSL/Git Bash через условные блоки |
| **Логирование** | Структурированный лог: `[2025-04-07 10:00:01] WARN | sleep (1234) | age=101h > 72h` |
| **Документация** | `README.md` с примером конфига и инструкцией по установке |

## Критерии готовности

- [ ] Скрипт запускается без ошибок на Ubuntu 22.04/24.04
- [ ] Уведомление приходит при процессе-долгожителе (например, `sleep 400000`)
- [ ] Игнор-лист работает
- [ ] Лог-файл создаётся и ротируется (в макс. версии)
- [ ] В режиме `--daemon` корректно реагирует на `Ctrl+C`
- [ ] (Максимум) Telegram-бот присылает сообщение
- [ ] (Максимум) Скрипт работает в Git Bash на Windows (упрощённо, без `notify-send`)
